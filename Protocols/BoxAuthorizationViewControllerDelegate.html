BoxAuthorizationViewControllerDelegate Protocol Reference

Conforms to NSObject
Declared in BoxAuthorizationViewController.h

Overview

The BoxAuthorizationViewControllerDelegate protocol defines methods that your delegate object must implement to interact with the authorization interface. The methods of this protocol notify your delegate when the authorization web view starts and stops loading content, or the user cancels the authorization operation.

The delegate methods are responsible for showing any progress UI (such as a UIActivityIndicatorView) and dismissing the controller when the operation is canceled. To dismiss the controller, call the dismissViewControllerAnimated:completion: method of the presenting controller responsible for displaying the BoxAuthorizationViewController object.

authorizationViewController:shouldLoadReceivedOAuth2RedirectRequest:

Asks the delegate whether the view controller should load the OAuth2 redirect URL in the web view.

- (BOOL)authorizationViewController:(BoxAuthorizationViewController *)authorizationViewController shouldLoadReceivedOAuth2RedirectRequest:(NSURLRequest *)request

Parameters

authorizationViewController

The authorization view controller object managing the authorization process

request

The OAuth2 redirect URL request which the authorization web view is requesting to load

Return Value

YES if the redirect URL should be loaded, NO if it should not be.

Discussion

Asks the delegate whether the view controller should load the OAuth2 redirect URL in the web view.

If the delegate does not implement this method, the default is YES.

If you wish to support iOS 5, you should implement this method and handle the OAuth2 redirect URL in it. Web views on iOS 5 do not support loading custom URL schemes.
